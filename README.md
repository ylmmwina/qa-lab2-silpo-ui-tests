# Звіт з лабораторної роботи №2
**Тема:** Автоматизоване UI-тестування веб-застосунку  
**Виконала:** Кірієнко Каріна  
**Об'єкт тестування:** [silpo.ua](https://silpo.ua/)

## Мета роботи
Набуття практичних навичок автоматизації тестування користувацького інтерфейсу веб-застосунків та ознайомлення з принципами побудови автоматизованих тестів.

## Стек технологій
* **Мова програмування:** TypeScript / JavaScript (Node.js)
* **Фреймворк для тестування:** Playwright
* **Браузер:** Google Chrome (Headed mode для обходу систем захисту Cloudflare)

## Опис реалізованих тестових сценаріїв
Під час виконання роботи було налаштовано середовище та написано 4 автоматизовані тести, які покривають ключові користувацькі сценарії:

1. **TC-00: Homepage loads successfully**  
   *Опис:* Перевірка базового завантаження головної сторінки та коректності заголовка (Title).
2. **TC-01: Search for an existing product**  
   *Опис:* Валідний пошук товару (наприклад, "Банан"). Перевіряється введення тексту, натискання Enter та видимість елементів з цінами (грн) у результатах видачі.
3. **TC-02: Search with invalid query**  
   *Опис:* Невалідний пошук. Перевіряється реакція системи на випадковий набір символів та поява повідомлення "нічого не знайдено".
4. **TC-10: Change interface language**  
   *Опис:* Перевірка локалізації. Скрипт знаходить перемикач мови, змінює її на англійську (EN) та перевіряє, чи змінилися ключові елементи інтерфейсу (наприклад, Cart, Search).

*Примітка: Локатори були оптимізовані (використання регулярних виразів та `.getByText`) для забезпечення стабільності тестів в умовах динамічної верстки та активного антибот-захисту (Cloudflare).*

## Інструкція із запуску (для перевірки)
Щоб запустити тести локально, виконайте наступні команди:
1. Встановіть залежності: `npm install`
2. Запустіть тести з відкритим браузером: `npx playwright test --headed`

## Висновок
У результаті виконання лабораторної роботи було створено базову структуру тестового проєкту на базі Playwright. Було успішно автоматизовано ключові UI-сценарії для реального e-commerce проєкту, реалізовано перевірки коректності (asserts) та забезпечено стабільність виконання тестів.
